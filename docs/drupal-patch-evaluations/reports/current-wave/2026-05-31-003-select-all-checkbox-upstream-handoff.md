## Critic Gate

The first implementation pass added `aria-label` while keeping `title`. Chrome exposed the same string as both accessible name and description, which was cleaner for axe but worse as a user-facing pattern. The final patch removes `title` and uses `aria-label` as the single programmatic name source.

The core JavaScript path is required because `/table` and Claro admin table-select markup can be generated by `core/misc/tableselect.js`, not only by Default Admin Twig templates. The Default Admin paths remain in scope because they hardcode the same checkbox for sticky table headers.

## Non-Claims

This does not claim a full table-select accessibility audit.

This does not claim human screen-reader, switch-control, or voice-control verification. The evidence is axe, DOM, FunctionalJavascript coverage, Chrome accessibility-tree output, and real keyboard activation.

This does not address the remaining `color-contrast` or `region` evaluator findings; those pre-existed and remain outside this scoped `label-title-only` patch.

The `/admin/people` evaluator case loaded but did not observe the baseline target violation in this runtime, so the actionable evaluator proof comes from `/admin/content` plus the deterministic `/table` smoke.
