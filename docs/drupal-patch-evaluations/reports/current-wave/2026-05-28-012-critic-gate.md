# Critic Gate: DRUPAL-A11Y-012 Empty Heading Elements

> Date: 2026-05-28
> Verdict: `ACCEPT`
> Reviewed artifact: `docs/drupal-patch-evaluations/patches/a11y-DRUPAL-A11Y-012-empty-heading-elements-codex-filter-tips-preprocess.patch`
> Updated: 2026-05-28 after regression test packaging

## Status Recommendation

`DRUPAL-A11Y-012-empty-heading-elements` can remain `VERIFIED` for the local patch candidate.

The refreshed patch is suitable to hand off for upstream review: it includes the filter-tips preprocess fix plus a focused kernel regression test for single-format and multi-format long filter tips rendered through Claro.

## Evidence Reviewed

- Baseline reproduced on `/dialog`, `/tabs`, and `/tabs/format/plain_text`.
- Candidate patch normalizes `filter_tips` variables through `FilterThemeHooks::preprocessFilterTips()`.
- Targeted `empty-heading` checks went to zero on all three affected routes.
- Default axe scans on affected routes showed no new rule families.
- Additional shared-surface smoke checks passed on `/admin/config/content/formats`, `/admin/config/content/formats/manage/basic_html`, and `/node/add/page`.
- PHP syntax check passed for `core/modules/filter/src/Hook/FilterThemeHooks.php`.
- PHP syntax check passed for `core/modules/filter/tests/src/Kernel/FilterTipsRenderTest.php`.
- Focused kernel test passed: `FilterTipsRenderTest`, 3 tests and 10 assertions.

## Findings

No blocking accessibility design findings.

The patch addresses the actual semantic failure: empty headings were not an authoring-content problem or a one-off theming-tools fixture issue. They were a template-data shape problem where the Twig template expected `tip.name` and `tip.list`, but the raw filter-tip buckets reached the template without a core preprocess normalizer.

The later revision correctly normalizes both the format-level wrapper attributes and per-item attributes, which matters because Claro and Default Admin call `tip.attributes.addClass(...)` and `item.attributes.addClass(...)`.

## Remaining Boundaries

- The full Drupal core test suite was not run.
- The current evidence is local DDEV/phpunit evidence, not maintainer acceptance.
- The remaining `region` and `heading-order` violations on the checked pages are pre-existing and unrelated, but they should be named in upstream evidence so the patch does not appear to claim full-page accessibility closure.

## Negative Space

This review is not claiming:

- that every route using `filter_tips` has been exhaustively tested;
- that the patch fixes `DRUPAL-A11Y-010` heading-order issues;
- that remaining `region` violations on affected pages are resolved;
- that upstream maintainers should accept the patch without a core-appropriate regression test.
